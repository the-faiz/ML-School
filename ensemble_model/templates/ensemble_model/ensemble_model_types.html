Types of Ensemble Models

In our previous discussion we have seen an overview of ensemble models. We essentially train multiple models ( called weak learners) and make final predictions by combining predictions from all the models. There are two things we can control here.

  • Training data

    We can keep the same training data for all the models or we can create different training samples for each of the models to train.

  • Model type

    We can select the same type of models ( for example all the models can be decision trees with different sets of hyperparameters ) or we can select different types of model. One can be decision tree others could be logistic regression, SVM, KNN etc.

We can generate four combinations by changing data or the model which are shown in diagram1.

  • Same Training data and same types of weak learners

    This combination is not useful as we will be training the same model on the same data. Essentially each model will be predicting the same thing for a given data point and hence predictions are not very useful in this case.

  • Same Training data and different model types

    This technique is called stacking. Weak learners can be any of the ML models such as decision tree , SVM , logistic regression, KNN etc. Details of stacking are discussed in the next section.

  • Different training data and same model types

    Bagging and boosting are the techniques which are trained in these settings. Random Forest is an example of bagging. Ada Boost & gradient boosting are the examples of boosting algorithms. We will discuss the details of each of the algorithms in upcoming discussions.

  • Different training data and different model types

    This is generally not used in practice. But readers can try out these settings.
